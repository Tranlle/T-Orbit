

项目名称
  基于屏幕选区与本地 OCR 的视频字幕实时翻译工具

  一、项目目标
  开发一个工具，对用户屏幕上指定区域内的现有视频字幕进行持续识别，并将识别结果实时翻译后显示出来。

  目标边界如下：

  - 只处理“画面中已经出现的字幕文字”
  - 不处理视频音频转字幕
  - 不依赖特定视频网站或播放器
  - 优先支持本地实时运行

  二、核心思路
  整体流程为：

  1. 用户框选视频字幕所在区域
  2. 系统按固定频率截取该区域画面
  3. 对截图进行预处理，提高字幕识别率
  4. 使用本地 OCR 识别字幕文字
  5. 对识别结果进行去重、稳定化处理
  6. 将文本翻译为目标语言
  7. 在悬浮层或独立区域实时显示原文与译文

  三、适用场景

  - YouTube、Bilibili、Netflix 等已显示字幕的视频
  - 在线课程、演讲、访谈、直播回放
  - 任意播放器窗口中的外挂字幕或内嵌字幕

  四、功能范围

  第一阶段：MVP

  - 手动框选字幕区域
  - 周期性截屏
  - 本地 OCR 识别
  - 字幕去重与更新判断
  - 实时显示原文
  - 实时显示译文
  - 开始 / 暂停 / 重新选区

  第二阶段：增强

  - 自动识别字幕区域
  - OCR 预处理参数调节
  - 双语显示模式
  - 历史字幕记录
  - 快捷键控制
  - 多语言翻译切换
  - 性能模式与精度模式切换

  五、技术方案概述

  输入层

  - 来源：用户手动框选的屏幕区域
  - 采样方式：定时截屏
  - 建议频率：每 300ms 到 800ms 采样一次

  识别层

  - 对截图进行图像预处理：
      - 灰度化
      - 对比度增强
      - 二值化
      - 降噪
      - 适度放大
  - 使用本地 OCR 引擎识别字幕文本

  文本处理层

  - 去除空白、噪声字符
  - 与上一条字幕做相似度比较
  - 避免同一句字幕重复刷新
  - 处理逐字出现或局部变化的问题

  翻译层

  - 将 OCR 输出送入翻译模块
  - 输出目标语言文本
  - 保留接口可替换性，后续可切换不同翻译引擎

  显示层

  - 悬浮窗显示
  - 支持只显示译文或双语显示
  - 支持低打扰、置顶、半透明样式

  六、关键难点与风险

  1. OCR 准确率受字幕样式影响较大
     风险来源：

  - 字号过小
  - 背景复杂
  - 字体发光、阴影、描边不规则
  - 低分辨率视频

  应对方式：

  - 增加图像预处理
  - 允许用户重新框选更精确的区域
  - 提供参数调节或识别模式切换

  2. 实时性与性能平衡
     风险来源：

  - 截图太频繁会提高 CPU 占用
  - OCR 和翻译串行处理会增加延迟

  应对方式：

  - 控制采样频率
  - 仅在画面明显变化时触发 OCR
  - 将截图、识别、翻译分阶段异步处理

  3. 字幕闪烁与重复更新
     风险来源：

  - 同一句字幕会连续出现多个相近版本
  - OCR 结果存在细微抖动

  应对方式：

  - 做文本相似度去重
  - 设置最短刷新间隔
  - 对短时结果做稳定化合并

  4. 翻译结果延迟或不稳定
     应对方式：

  - 先保证原文识别稳定
  - 翻译模块独立，允许降级只显示原文

  七、开发里程碑

  1. 原型验证

  - 完成屏幕选区
  - 完成定时截屏
  - 完成本地 OCR 输出控制台文本
  - 验证常见字幕场景下可识别性

  2. MVP 开发

  - 加入图像预处理
  - 加入去重与稳定逻辑
  - 加入实时显示界面
  - 加入翻译模块
  - 实现完整闭环

  3. 可用性优化

  - 调整刷新策略
  - 降低资源占用
  - 增加快捷键和设置项
  - 优化显示样式与交互

  4. 测试与发布准备

  - 不同网站和播放器测试
  - 不同字幕样式测试
  - 异常处理与日志补充
  - 打包发布

  八、验收标准
  满足以下条件可视为首版可用：

  - 用户可手动选择字幕区域
  - 工具能稳定识别该区域内字幕
  - 相同字幕不会频繁重复刷新
  - 能在 1 到 2 秒内显示译文
  - 普通视频场景下可持续运行，不出现明显卡顿

  九、结论
  该项目方向明确、实现路径可控，适合作为一个先做小闭环、再逐步增强的产品。对于“翻译视频里已经显示出来的字幕”这一目标，采
  用“屏幕选区 + 本地 OCR + 实时翻译显示”的方案是可行的，且具备较好的通用性与落地性。
